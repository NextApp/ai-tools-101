# 流量增长审计框架

这是一套可复用的内容站流量增长决策框架，每当你需要评估一个站的流量潜力或制定增长策略时，按这个逐项走一遍。

## 第一步：竞争格局反查

常规做法是"找低竞争关键词然后写文章"——这太慢。你要反着来：

1. 打开站点的现有文章列表，按类型分类（Best 列表 / vs 对比 / How-to 教程 / Review / Guide）
2. 对每类文章，手动 Google 搜索 3 个代表性标题，看第一页是谁在霸占
3. 记录每个 SERP 的对手：是 Reddit（DA 91）？是 G2/Capterra？是 TechRadar/PCMag？
4. **如果第一页 5 个以上结果来自 DA 80+ 站点 → 这个内容类型对你来说是死路，不要继续投产**
5. **如果第一页有 Reddit/Quora 但没有专业站 → 这是你的机会窗口**

## 第二步：内容类型缺口

对着站点的文章列表跑这个检查：

| 维度 | 检查项 |
|------|--------|
| 格式多样性 | Best 列表 / vs 对比 / How-to / Review / Guide / Opinion — 各占比多少？ |
| 搜索意图覆盖 | 信息型（"what is X"）/ 导航型（"X login"）/ 交易型（"best X for Y"）/ 商业型（"X review"）— 哪种缺了？ |
| 品类覆盖 | 如果有工具/产品数据库，哪些品类有工具但没文章？ |
| 漏斗深度 | 顶部（入门指南）→ 中部（对比评测）→ 底部（具体教程）— 缺哪一层？ |

**规则：新站不应该把 50% 以上的产能投在同一种内容类型上。** 如果你发现 11/25 篇都是 "Best X 列表"，立即停止写这类文章，产能转向缺口类型。

## 第三步：程序化 SEO 可行性

如果你的站点有任何结构化数据（工具库、产品列表、课程目录），你必须回答这三个问题：

1. **有没有 N×M 的组合可以生成页面？** 例：50 个工具 × compare_with 关系 = 200-500 个 "A vs B" 页
2. **有没有分类/标签页面可以加长内容变成独立 landing page？** 每个分类页应该至少有 500 字的原创介绍
3. **有没有常见的查询模板可以自动化？** 例："[工具名] pricing"、"[工具名] alternatives"、"[工具名] vs [工具名]"

程序化 SEO 的核心逻辑：**写一次模板 → 生成几百个页面 → 每个页面捕获 10-50 次/月的长尾搜索。加起来就是几千到几万次月搜索覆盖。** 这比写 10 篇新文章的 ROI 高得多。

## 第四步：非搜索流量渠道

内容站最危险的状态就是 100% 依赖 Google 搜索。你必须检查：

- **是否有 RSS feed？** — 分发到 Feedly、Newsletter 聚合器
- **是否有 Pinterest 策略？** — AI 教程天然适合，流量寿命是 Google 的 100 倍
- **是否有可下载资源？** — PDF cheatsheet / Google Sheets 模板 / prompt 模板包 → 最高效的 backlink 磁铁
- **是否有邮件列表入口？** — 至少放一个 ConvertKit/Substack 链接
- **是否有社交媒体分发？** — X/LinkedIn/Reddit 有没有发布计划？

**规则：至少要有 2 条非搜索流量渠道在运转。** 单一渠道的站点在 Google 一次算法更新后可能直接归零。

## 第五步：结构化数据盲区

对着站点的 JSON-LD 检查：

- 有 Article schema 吗？→ 博客文章页
- 有 SoftwareApplication/Product schema 吗？→ 工具/产品页
- 有 FAQ schema 吗？→ 工具页加 5 条 FAQ 就能直接吃 Google "People also ask" 入口
- 有 HowTo schema 吗？→ 教程文章加步骤 JSON-LD
- 有 BreadcrumbList 吗？→ 辅助 Google 理解站点结构
- 有 Organization schema 吗？→ 站点级别的品牌信号

**每个缺的 schema 类型都是一个未被占用的 SERP 特殊展示位。**

## 第六步：EEAT 信号检查

Google 对内容站的权威性评估看这些：

- 文章有没有引用外部权威来源（研究机构、官方文档、行业报告）？
- 每篇文章有没有 <2 条外链就是红色警报
- 有没有作者信息展示？
- 有没有"最后更新日期"展示 + dateModified schema？
- 有没有关于我们页面和联系方式？

## 第七步：变现对齐检查

流量和变现必须一起考虑，而不是分开：

- 检查站点当前的变现方式：AdSense？联盟链接？自营产品？
- 对每篇文章做变现最大化检查：联盟链接是否与文章内容匹配？CTA 是否明显但不突兀？
- **规则：不要为了流量写和变现无关的内容。** 如果你写 "DeepSeek 教程" 但 DeepSeek 没有 affiliate program，你应该写 "DeepSeek vs ChatGPT" 然后在 ChatGPT 那段放联盟链接。

## 输出格式

每次审计完成后，按以下格式输出结论：

```
## 增长审计：[站点名]

### 不该做的事（停止投产）
- [具体内容类型/话题] — 因为 [SERP 竞争/已过度覆盖/ROI 低]

### 该做的事（按 ROI 排序）
1. [最高杠杆行动] — 工作量 [X]，预期流量 [Y]，周期 [Z]
2. ...

### 关键数字
- 当前文章数：[N]，目标：[M]
- 缺口的流量渠道：[列表]
- 下次审计时间：[日期]
```

## 使用示例

执行时加载此 skill，然后跑：
```bash
python3 对站点进行文章审计的脚本
```
根据审计结果走完七步框架，输出结论。
